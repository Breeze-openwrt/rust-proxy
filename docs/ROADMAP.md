# 🗺️ Rust-Proxy 演进路标 (Roadmap)

> **目标**: 成为性能无可争议的 Linux 代理标杆。

---

## 📅 阶段一：地基与暴力提速 (已达成)
- [x] **Monoio 核心**: 基于 `io_uring` 的 Thread-per-core 异步引擎。
- [x] **eBPF 重定向**: 核心重定向逻辑下沉至内核 SockMap。
- [x] **物理调优**: 自动应用 TCP_QUICKACK 与 4MB 巨型窗口。
- [x] **高性能发布**: CI 支持 x86-64-v3 (AVX2) 自动分层构建。

## 🚀 阶段二：动态优化与弹性转发 (进行中)
- [ ] **JumpStart 后端预热**: 实现毫秒级的后端连接池预热，消除首包延迟。
- [ ] **动态 BPF 映射**: 支持在不重启服务的前提下，通过 CLI 动态更新分流路由规则。
- [ ] **流控暴力对冲**: 智能缓冲区管理，确保在千兆对冲环境下不发生内存膨胀。

## 🌟 阶段三：智能感知与生态扩展 (长远目标)
- [ ] **AI 流量指纹识别**: 基于 eBPF 数据采样，实现无感知的协议智能分流。
- [ ] **多内核协议栈适配**: 扩展支持 QUIC/UDP 加速。
- [ ] **可视化观测系统**: 提供基于 Grafana 的内核态实时延迟波形图。

---

> [!TIP]
> **AI 参考指引**: 当项目处于不同发展阶段时，应优先分配算力或逻辑复杂度到当前阶段的核心目标上。
